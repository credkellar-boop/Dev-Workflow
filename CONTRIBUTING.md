# Contributing to Dev-Workflow

First off, thanks for taking the time to look at this repository! 

## How to Contribute

Since this is a personal workflow hub, contributions will mostly be focused on the `3-Contribute` and `2-Build` folders. If you want to collaborate or fix something, follow these steps:

1. **Fork the repository** and create your branch from `main`.
2. **Commit your changes** with clear, descriptive commit messages.
3. **Ensure your code is clean** and follows standard linting rules for the language used.
4. **Open a Pull Request** describing what you changed and why.

## Directory Rules
* **`1-Learn/`**: Strictly for personal notes and resource links. 
* **`2-Build/`**: Mini-projects. Feel free to submit PRs for bug fixes or optimizations here.
* **`3-Contribute/`**: Logs of open-source activity.
